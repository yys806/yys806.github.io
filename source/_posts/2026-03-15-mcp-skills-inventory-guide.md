这篇是我的 MCP 和 Skills 实战清单。后续新增能力会持续补到这里，避免信息分散。

## 我现在的 Skills（含新增）

### 本地安装 Skills

- `yt-dlp-downloader`：多平台视频下载、提取音频、字幕处理。
- `xlsx`：表格处理、公式、分析与格式整理。
- `pdf`：PDF 提取、拆分、合并、表单处理。
- `docx`：Word 文档编辑、修订、批注与格式保留。
- `pptx`：演示文稿生成和改版。
- `paper-2-web`：论文转网页/海报/视频的传播链路。
- `frontend-design`：高质量前端页面与视觉实现。
- `find-skills`：按任务检索可用技能。
- `skill-creator`：创建和迭代 skill 的标准流程。
- `planning-with-files`：复杂任务文件化计划管理。
- `netlify-auto-deploy`：Netlify 自动部署。
- `paper-experiment`：论文中的实验助手。
- `paper-literature` ：论文中的写作助手。

### 内置 Skills

- `playwright`：浏览器自动化与回归验证。
- `frontend-ui-ux`：UI/UX 设计实现。
- `git-master`：Git 提交、历史追溯、分支整理。
- `dev-browser`：持久化浏览器交互自动化。

## 我常用的 MCP（按功能）

- `GitHub MCP`：仓库检索、代码搜索、Issue/PR 协作、Review 评论、合并与跟踪。
- `Chrome DevTools MCP`：页面快照、网络请求排查、性能分析、Lighthouse 质检。
- `Puppeteer MCP`：自动化浏览器交互、截图、抓取和流程回放。
- `Stripe MCP`：客户、产品、价格、支付链接、发票、订阅、退款等支付链路。
- `Neon MCP`：数据库项目/分支、SQL 执行、迁移、调优与 Schema 对比。
- `Semgrep MCP`：SAST/SCA 扫描、自定义规则、历史问题回看。
- `Context7 MCP`：库文档与示例检索，减少 API 用法试错。
- `Figma MCP`：设计稿节点信息读取与素材导出。
- `Replicate MCP`：图像生成、变体和 SVG 输出。
- `Web Search MCP`：全网最新信息检索，补齐外部上下文。
- `Webfetch MCP`：指定 URL 转 Markdown/Text/HTML 做归档和引用。

## 我现在的使用原则

1. 涉及“最新消息”的任务，先检索，再输出结论。
2. 需要发布的内容先给草稿确认，再执行发布。
3. 自动化流程优先固化成 skill，避免重复手工操作。
