我的MCP与Skills清单说明

🧭 为什么要做这份清单

最近我把本机常用的 Skills 和 MCP 做了一次系统盘点。目的很简单：把“会用”升级成“稳定复用”。

这篇是我的公开版说明文档：写给未来的自己，也写给同样在搭 AI 工作流的人。

来源：skills.sh

🔧 我现在在用的 Skills（本地）

当前本机保留并高频使用的技能有这些：

  • XiaohongshuSkills：小红书图文/视频发布、登录检查、内容检索与互动
  • yt-dlp-downloader：下载视频、抽音频、字幕处理
  • blog：Hexo 博客写作与发布
  • website-organizer：WebStack-Hugo 导航站维护
  • xlsx / pdf / docx / pptx：文档和表格生产力处理
  • paper-2-web：论文转网页/海报/视频流程
  • frontend-design：前端页面和视觉实现
  • find-skills / skill-creator：技能搜索与技能构建
  • planning-with-files：复杂任务规划落盘

什么时候优先用 Skill？

如果一个任务满足下面两条，就值得固化成 Skill：

  1. 重复频率高（每周都要做）
  2. 步骤固定（可标准化）

🧠 我现在常用的 MCP 能力（按场景）

来源：modelcontextprotocol/docs

1) 开发协作：GitHub MCP

  • 查仓库、查代码、提 PR、看 Review、回评论
  • 适合把“代码协作”流程化

来源：modelcontextprotocol/servers

2) 代码与代理工作流：Claude Code 生态

  • 适合做命令式开发、脚本化修改、自动验证

来源：anthropics/claude-code

3) 数据库自动化：Neon MCP

  • 项目/分支管理、SQL 执行、迁移验证、性能排查

来源：neondatabase/neon

✅ 我当前的落地原则

为避免“越自动化越混乱”，我固定了三条规则：

  1. 先验证后执行：发布、迁移、删除都要先检查
  2. 强约束可追溯：关键动作必须有日志与结果状态
  3. 保留人工确认闸门：自动化不等于无审核

📝 给未来自己的操作清单

  • 每月更新一次 Skills/MCP 清单
  • 新增能力先试运行 3 次，再决定是否固化
  • 对外输出时，把“做法 + 约束 + 失败恢复”一起写清楚

如果你也在做个人 AI 工作台，建议从一个高频场景开始（例如“内容发布”或“代码协作”），先跑通，再扩展。
